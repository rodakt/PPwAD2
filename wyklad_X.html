Metody specjalne

W Pythonie metodami specjalnymi (albo magicznymi) nazywamy te metody, ktrych nazwy rozpoczynaj i kocz dwa podkrelniki np.: __init__(), __str__(), __eq__() i wiele innych. Spisy metod specjalnych znajdziesz tutaj i tutaj. Wyjtkowo metod specjalnych polega na tym, e:

  1. s wywoywane niejawnie przez odpowiedni skadni;
  2. konwencja narzuca, za co kada z tych metod jest odpowiedzialna.

Poznalimy ju metod __init__() - jeli klasa t metod posiada, to jest ona automatycznie wykonywana podczas tworzenia obiektw klasy. Jej cel to inicjalizacja obiektu, czyli ustawienie w obiekcie atrybutw i ich startowych wartoci.

Podamy teraz kilka przykadw klas z rnymi metodami specjalnymi.

Funkcja liniowa

Klasa FunkcjaLiniowa reprezentuje znan ze szkoy funkcj liniow $$ y=ax+b $$

In[1]:
class FunkcjaLiniowa:
    
    def __init__(self, a, b):
        self.a, self.b = a, b
    
    def __str__(self):
        if self.b >= 0:
            return 'y = {}*x + {}'.format(self.a, self.b)
        return 'y = {}*x - {}'.format(self.a, -self.b)
    
    def __repr__(self):
        return 'FunkcjaLiniowa({}, {})'.format(self.a, self.b)
    
    def __call__(self, x):
        return self.a*x + self.b

Oto funkcja $y=2x-3$:

In[2]:
f = FunkcjaLiniowa(2, -3)

Notatnik odwouje si do metody __repr__(), aby przedstawi obiekt:

In[3]:
f
Out[3]:
FunkcjaLiniowa(2, -3)

Funkcja print() poszukuje metody __str__():

In[4]:
print(f)

y = 2*x - 3

Operator wywoania () odwouje si do metody __call__():

In[5]:
f(5)
Out[5]:
7

Metody __repr__() i __str__() powinny zwraca acuch i nie powinny mie parametrw innych ni self. W obu przypadkach zwracany acuch stanowi opis obiektu. Metoda __str__() powinna przedstawia obiekt w sposb nieformalny i wygodny dla czowieka. Z kolei metoda __repr__() prezentuje oficjaln i formaln reprezentacj obiektu. Warto zwracana przez __repr__() powinna by, o ile to moliwe, poprawnym wyraeniem Pythona.

Spjrz jak prezentowany jest obiekt daty i czasu w module datatime:

In[6]:
import datetime

dzisiaj = datetime.datetime.today()

Formalne przedstawienie metod __repr__():

In[7]:
dzisiaj
Out[7]:
datetime.datetime(2025, 5, 14, 8, 39, 7, 810834)

Format przyjazny czowiekowi zwracany przez __str__():

In[8]:
print(dzisiaj)

2025-05-14 08:39:07.810834

Rnica midzy tymi metodami zostaa wyczerpujco opisana w tym wpisie na StackOverflow.

Metoda __call__() jest poszukiwana przez operator wywoania (). Argumenty operatora wywoania zostaj przekazane do metody __call__() (ktra jako pierwszy argument musi mie self). Jeli obiekt posiada t metod, to staje si obiektem wywoywalnym. Fakt ten potrafi wykry funkcja callable():

In[9]:
callable(f)
Out[9]:
True

Poznalimy wic ju niemal wszystkie kategorie obiektw wywoywalnych Pythona:

  • funkcje wbudowane, np. len();
  • funkcje zdefiniowane przez uytkownika z pomoc def lub lambda;
  • metody wbudowane, np. metoda ancuchw upper();
  • klasy;
  • metody klasy zdefiniowane przez uytkownika;
  • obiekty posiadajce metod __call__().

Przykad: rniczkowanie numeryczne

Pochodn funkcji $f\colon (a, b)\to\mathbb{R}$ w punkcie $x_0\in (a, b)$ nazywamy granic ilorazu rnicowego

$$ f'(x_0)=\lim_{h\to 0}\frac{f(x_0+h) - f(x_0)}{h}. $$

Jeeli granica ta istnieje, to mwimy, e $f$ jest rniczkowalna w $x_0$.

Nasze podejcie do rniczkowania numerycznego bdzie polegao na tym, e zamiast liczy powysz granic, obliczymy sam iloraz rnicowy dla $h$ konkretnego i bardzo bliskiego zera.

Pochodna numeryczna

Klasa Pochodna przyjmuje parametry:

  • f -- obiekt wywoywalny zwracajcy liczby. Z punktu widzenia uytkownika jest to funkcja rzeczywista.
  • h -- "dostatecznie bliska zeru" warto liczbowa. Moe by ujemna lub dodatnia. Domylnie 1e-7 czyli $10^{-7}$.

Na podstawie tych parametrw klasa tworzy obiekt wywoywalny przybliajcy pochodn f.

In[10]:
class Pochodna:
    
    def __init__(self, f, h=1e-7):
        self.f = f
        self.h = h
    
    def __call__(self, x):
        f, h = self.f, self.h
        return (f(x + h) - f(x)) / h

Wektory

Klasa Wektor reprezentuje wektory w przestrzeni euklidesowej:

In[23]:
class Wektor:
    
    def __init__(self, *wsp):
        self.wsp = wsp
    
    def __repr__(self):
        return 'Wektor({})'.format(', '.join(str(x) for x in self.wsp))
    
    def __add__(self, w):
        if len(self.wsp) != len(w.wsp):
            raise ValueError('Konflikt wymiarw.')
        
        wynik_wsp = tuple(x + y for x, y in zip(self.wsp, w.wsp))
        return Wektor(*wynik_wsp)
    
    def __rmul__(self, skalar):
        wynik_wsp = tuple(skalar * x for x in self.wsp)
        return Wektor(*wynik_wsp)
    
    def __neg__(self):
        return -1 * self
    
    def __sub__(self, w):
        return self + -w
    
    def __eq__(self, w):
        if len(self.wsp) != len(w.wsp):
            return False
        
        return all(x == y for x, y in zip(self.wsp, w.wsp))

Metoda __add__() przecia operator +.

Jeli x i y s obiektami i x posiada metod __add__(), to wyraenie

x + y

jest rwnowane z

x.__add__(y)

Oto zestaw metod dziaajcych analogicznie do __add__():

  • __add__(self, y) -- dodawanie +;
  • __sub__(self, y) -- odejmowanie -;
  • __mul__(self, y) -- mnoenie *;
  • __matmul__(self, y) -- mnoenie macierzowe @;
  • __truediv__(self, y) -- dzielenie zmiennoprzecinkowe /;
  • __floordiv__(self, y) -- dzielenie podogowe //;
  • __mod__(self, y) -- reszta z dzielenia %;
  • __divmod__(self, y) -- divmod();
  • __pow__(self, y[, modulo]) -- potga **;
  • __lshift__(self, y) -- przesunicie bitowe w lewo o y pozycji <<;
  • __rshift__(self, y) -- przesunicie bitowe w prawo o y pozycji >>;
  • __and__(self, y) -- koniunkcja bitowa &;
  • __xor__(self, y) -- bitowa alternatywa rozczna ^;
  • __or__(self, y) -- alternatywa bitowa |.

Zauwa, e klasa Wektor nie ma metody __mul__(), posiada za to __rmul__(). Dlaczego?

Jeli x, y s obiektami i x nie ma metody __mul__() lub x.__mul__(y) jest niewykonalne, to wyraenie

x * y

wywouje

y.__rmul__(x)

Operacja x.__mul__(y) jest uznawana za niewykonaln, gdy wartoci zwracan przez to wywoanie jest warto specjalna NotImplemented. Zobacz szczegy tutaj i tutaj.

W klasie Wektor metoda __rmul__() implementuje mnoenie liczby i wektora, a nie wektora przez wektor. Wyraenie

2 * w

prbuje najpierw wywoania self.__mul__(w), gdzie self jest dwjk. To mnoenie si nie powiedzie, gdy metoda .__mul__() w klasie liczb cakowitych nie potrafi mnoy liczb przez obiekty klasy wektor. Wtedy interpreter poszukuje metody __rmul__() w obiekcie w. Gdy j znajdzie, wywouje

w.__rmul__(2)

co daje podany efekt. Nie powiedzie si jednak

w * 2

gdy w nie posiada metody __mul__()!

Moglibymy j dopisa:

class Wektor:
    
    <...> 
    
    def __rmul__(self, skalar):
        wynik_wsp = tuple(skalar * x for x in self.wsp)
        return Wektor(*wynik_wsp)
    
    def __mul__(self, skalar):
        return skalar * self
    
    <...>

Wyraenie skalar * self odwoa si do metody __rmul__(), nie ma potrzeby pisania po raz drugi tego samego kodu.

Jednak w algebrze liniowej mnoenie wektora przez skalar odbywa si w ten sposb, e skalar stoi z lewej strony wektora. Nie ma nic zego w tym, e operacja 2 * w jest wykonalna a w * 2 nie.

Oto metody dziaajce podobnie do __rmul__():

  • __radd__(self, x)
  • __rsub__(self, x)
  • __rmul__(self, x)
  • __rmatmul__(self, x)
  • __rtruediv__(self, x)
  • __rfloordiv__(self, x)
  • __rmod__(self, x)
  • __rdivmod__(self, x)
  • __rpow__(self, x)
  • __rlshift__(self, x)
  • __rrshift__(self, x)
  • __rand__(self, x)
  • __rxor__(self, x)
  • __ror__(self, x)

Metody __eq__(), __ne__() przeciaj operatory porwnywania == i !=. Jeli nie zwracaj bezporednio True/False, to na wartoci zwracanej wywoywane jest bool(). Metody __ne__() nie trzeba pisa - tworzona jest automatycznie, wywouje __eq__() i zwraca zaprzeczenie.

Zestaw metod porwnujcych:

  • __lt__(self, y) -- <
  • __le__(self, y) -- <=
  • __eq__(self, y) -- ==
  • __ne__(self, y) -- !=
  • __gt__(self, y) -- >
  • __ge__(self, y) -- >=
